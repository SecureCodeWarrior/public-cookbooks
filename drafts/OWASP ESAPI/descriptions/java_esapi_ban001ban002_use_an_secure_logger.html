Abstract

Out of Java best practices and coding guidelines, recommendations where abstracted that recommend the usage of OWASP Enterprise Security API (ESAPI). Only authorized routines from that library should be used for logging. As such, developers are discouraged from printing stack traces or using System.out routines or standard Java routines for logging.

Description

It is not recommended to use the standard java implementation for output or logging as there is a more secure OWASP ESAPI alternative. More specific, it is recommended to usa a Logger that is an implementation of org.owasp.esapi.Logger. One commonly used implementing class of this interface is the Log4JLogger.

Correct code example:

import org.owasp.esapi.reference.Log4JLogger;
...
Log4JLogger logger = Log4JLogger.getInstance(getClass());
logger.log(org.apache.log4j.Level.ALL,"My log message");
