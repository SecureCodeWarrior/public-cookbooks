Abstract

Out of Java best practices and coding guidelines, recommendations where abstracted that recommend the usage of OWASP Enterprise Security API (ESAPI). Only authorized routines from that library should be used for for for encoding text.

Description

It is not recommended to use the standard java implementation for encoding text as there is an OWASP ESAPI alternative. Concrete this means that the java.net.URLEncoder class is banned. More specific, one should use an implementation of the Encoder interface, it provides an alternative with the same methods as the original Class but guarantees cryptographic security. An implementing class of this interface is the DefaultEncoder.

Correct code example:

import org.owasp.esapi.reference.DefaultEncoder;
...
String encoded = DefaultEncoder.getInstance().encodeForURL(string);
