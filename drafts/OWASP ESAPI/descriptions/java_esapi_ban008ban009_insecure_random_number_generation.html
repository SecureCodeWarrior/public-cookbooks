Abstract

Out of Java best practices and coding guidelines, recommendations where abstracted that recommend the usage of OWASP Enterprise Security API (ESAPI). Only authorized routines from that library can be used for random number generation. As such, developers can not use standard java implementations to generate random numbers.

Description

It is not recommended to use the standard java implementation for random number generation as there is an OWASP ESAPI alternative. Specifically banned methods are:
  • BAN008a: Math.Random.nextInt, nextFloat, nextLong, nextDouble, nextBoolean, nextBytes: insecure Random number generation
  • BAN008b: Math.random: uses the Math.Random class methods
  • BAN009: File.createTempFile: generates random file names insecurely
As an alternative it is recommended to use an implementation of the Randomizer interface, it provides methods for generating several data types as well as filenames and GUID. An Implementing class of this interface is the DefaultRandomizer.

Correct code example:

import org.owasp.esapi.reference.DefaultRandomizer;
...
float random = randomizer.getRandomReal(0,1);
