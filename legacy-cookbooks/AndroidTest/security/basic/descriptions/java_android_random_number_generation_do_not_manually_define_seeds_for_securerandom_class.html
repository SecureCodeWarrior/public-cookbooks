Abstract

Out of best practices and Android coding guidelines, recommendations were abstracted which state that you should not manually define seeds when generating random numbers.

Description

Use the SecureRandom class and let the default implementation generate its own seed.

Correct code example:

SecureRandom random = new SecureRandom();
byte bytes[] = new byte[20];
random.nextBytes(bytes);

Violating this guideline can cause
