Abstract

Secure coding practices prescribe to set the <transport-guarantee> security constraint for all URLs to CONFIDENTIAL.

Description

SSL ensures that third parties can not access data in transit including sensitive and private data. In addition, SSL ensures the data can not be changed during transmission. Adding a security constraint to the web.xml file achieves this. The <transport-guarantee> value should be set to CONFIDENTIAL. It is recommended to do this for the entire website. To do this, set the <url-pattern> to include every url, with the /* wildcard.

Correct code example:

<web-app>
   ...
   <security-constraint>
      <web-resource-collection>
         <web-resource-name>Entire Website</web-resource-name>
         <url-pattern>/*</url-pattern>
      </web-resource-collection>
      <user-data-constraint>
         <transport-guarantee>CONFIDENTIAL</transport-guarantee>
      </user-data-constraint>
   </security-constraint>
   ...
</web-app>
