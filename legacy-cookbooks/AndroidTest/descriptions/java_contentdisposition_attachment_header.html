Abstract

Secure coding practices prescribe to set the Header Content-Disposition to include "attachement".

Description

It is recommended to add the "Content-Disposition: Attachment" header to the response of static files, as this will secure the website against Flash or PDF-based cross-site content-hijacking attacks. It is recommended that this practice be performed for all files that users need to download in all modules that deal with file downloads.

Correct code example:
response.setHeader("Content-Disposition","attachment; filename=\"" + file + "\"");

Violating this guideline can cause

Web vulnerabilities
