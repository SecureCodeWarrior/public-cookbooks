Abstract

Out of best practices and Android coding guidelines, recommendations were abstracted which state that SQLCipher should be preferred over SQLite databases .

Description

SQLCipher offers a fully-encrypted database. It is very easy to implement in your application as SQLCipher offers their own implemenation of the SQLite methods. A downside of SQLCipher is a slight performance decrease, as well as the need to securely store the encryption key. Thus, SQLCipher can be used if the key can be supplied by the user of the application (e.g., a pin or password), or if the key can be securely sent over the network to the application. Never store the encryption key in the application itself!
