Abstract

Secure coding practices prescribe to check user input against illegal characters before creating XPATH queries.

Description

Input used into an XPATH expression must not contain any of the following characters:

   ( ) = ' [ ] : , * / WHITESPACE

When user input is used to create XPATH queries, the input must always be checked against these characters. For example, one can check if the input contains any of these characters by using a regex.

Correct code example:
if (expression.matches(".*(\\(|\\)|=|'|\\[|\\]|:|,|\\*|\\/| |\\t|\\n|\\r).*")){
   //handle invalid input
}
javax.xml.xpath.XPathExpression expr = xpath.compile(expression);

Using string concatenation without input validation/sanitization to construct an XPath Query can lead to an XPath Injection vulnerability.
