Abstract

Out of best practices and android coding guidelines, recommendations were abstracted which state that the SecureRandom class should be used instead of the Random class.

Description

The Random class does not return cryptographically secure tokens. Using the SecureRandom class secure tokens can be generated.

Correct code example:

The next method should receive an integer between 0 and 32. 
SecureRandom generator = new SecureRandom();
int numBits = 32; 
int token = generator.next(numBits);
