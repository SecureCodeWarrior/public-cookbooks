Abstract

Secure coding practices prescribe to print error messages instead of stack traces.

Description

It is recommended not to print stack traces as this will leak information about the code to malicious users. Instead use proper error-handling and display less descriptive error messages to the user.

Correct code example 
try {
    ...
} catch (Exception e){
    System.err.println("Something went wrong with ...");
}
