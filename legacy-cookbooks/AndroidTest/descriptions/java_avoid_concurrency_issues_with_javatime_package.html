Abstract

Secure coding practices prescribe that the time package in Java JDK8 should be used for concurrency.

Description

To handle time and dates the java.time package is recommended. The older classes (such as java.util.Date and SimpleDateFormatter) are not thread-safe, leading to potential concurrency issues. On top of that there are some inconsistencies when using these older APIs.

Class information:

package java.time.LocalDateTime;
   public static LocalDateTime now();

Correct code example:

LocalDateTime date = LocalDateTime.now();
