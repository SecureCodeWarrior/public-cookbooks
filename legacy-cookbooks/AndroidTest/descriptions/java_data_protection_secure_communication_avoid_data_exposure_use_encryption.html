Abstract

Secure coding practices prescribe to use symmetric encryption for secure data storage.

Description

It is important to protect any sensitive user information that is communicated by the application. In order to do this the communicated data should be properly encrypted. For communicating large chunks of data it is recommended to use a symmetric encryption algorithm. Symmetric encryption should be used for bulk encryption, i.e. to store sensitive data securely or to encrypt communication after a secure channel has been established.
The recommended algorithm for communicating large chunks of data is is AES, used in GCM mode with no padding.

Correct code example:

public static byte[] encryptForLocalStorage(byte[] plainText, byte[] IV, Key key) throws Exception {
    Cipher c = Cipher.getInstance("AES/GCM/NoPadding");
    GCMParameterSpec gcmSpec = new GCMParameterSpec(GCM_TAG_LENGTH * 8, IV);
    c.init(Cipher.ENCRYPT_MODE, key, gcmSpec);
    return c.doFinal(plainText);
}
