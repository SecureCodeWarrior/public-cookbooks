Abstract

Out of best practices and Android coding guidelines, recommendations were abstracted which state that storing sensitive information in device encrypted storage is unsafe.

Description

Device Encrypted storage should not be used to store sensitive information. It's a less secure storage location than the Credential Encrypted Storage. This latter location is only available after the user has unlocked their device. The Credential Encrypted Storage is the default storage location, so no extra configuration is needed.

Correct code example:

String filename = "myfile";
String string = "output";
FileOutputStream outputStream;
try {
   outputStream = contextObject.openFileOutput(filename, Context.MODE_PRIVATE);
   outputStream.write(string.getBytes());
} catch (Exception e) {
   e//handle Exception
} finally {
   outputStream.close();
}
