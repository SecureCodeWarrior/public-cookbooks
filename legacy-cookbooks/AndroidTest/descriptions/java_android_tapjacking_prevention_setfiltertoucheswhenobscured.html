Abstract

Android best practices and secure coding guidelines recommend to always set the filterTouchesWhenObscured flag for Views to true.

Description

Failing to filter touches when obscured allows for an attacker to attempt tapjacking. To prevent this, all View elements should explicitely set filterTouchesWhenObscured to true. This can be done in Java code by using the method setFilterTouchesWhenObscured or by setting the attribute android:filterTouchesWhenObscured to true in the XML layout file .

Code example to prevent tapjacking in a layout XML file:

<Button 
   android:filterTouchesWhenObscured="true" />

Code example to prevent tapjacking in Java:

Button safeButton = (Button)findViewById(...); 
safeButton.setFilterTouchesWhenObscured(true);
