Abstract

Secure coding practices prescribe to log error messages instead of stack traces.

Description

It is recommended not to print stack traces as this will leak information about the code to malicious users. Instead it is recommended to log errors using a safe logger.

Correct code example:

import org.slf4j.Logger;
...
public void setTemperature(Integer temperature) {
    oldT = t;
    t = temperature;
    logger.debug("Temperature set to {}. Old temperature was {}.", t, oldT);
    if(temperature.intValue() > 50) {
        logger.info("Temperature has risen above 50 degrees.");
    }
}
