Abstract

Secure coding practices prescribe to avoid using the method getParameter() to retrieve parameters from requests.

Description

Retrieving information from the request should happen through secured API's. A class GetParameterHelper was introduced which implements a couple dozen API's to retrieve information from the request. The following functions are available to retrieve information from requests securely. Additional approved API's can be found in the class GetParameterHelper itself.

Class information:

import com.senseisecurelibs.GetParameterHelper;
    public static double getParameterAsDoubleSecured(HttpServletRequest request, String key);
    public static int getParameterAsIntegerSecured(HttpServletRequest request, String key);
    public static boolean getParameterAsBooleanSecured(HttpServletRequest request, String key);
    public static String getParameterAsStringSecured(HttpServletRequest request, String key);

Correct code example:

double data= GetParameterHelper.getParameterAsDoubleSecured(request, "balance");
