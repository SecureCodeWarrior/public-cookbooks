Abstract

Secure coding practices prescribe to avoid running OS commands from the program.

Description

It is not recommended to use the java Runtime.exec(String) method. The overloaded method Runtime.exec(String[]) is secure as long as no user input is used in the first String in the array. However, it is best practice to avoid running OS commands from the program altogether. Use API calls whenever possible.
