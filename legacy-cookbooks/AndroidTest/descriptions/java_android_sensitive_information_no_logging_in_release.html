Abstract

Out of Android best practices and secure coding guidelines, recommendations were abstracted to avoid Logging in releases.

Description

Logging can be prevented by:
  • Use ProGuard to delete the Log class methods in release mode
  • Make use of if (BuildConfig.DEBUG) before every log entry, in order to avoid log files in release mode and set debuggable to false in release mode
  • Avoid all methods of the Log class alltogether

Violating this guideline can cause

Mobile vulnerabilities

  • Unintended Data Leakage - Logging Sensitive Information. Learn more
