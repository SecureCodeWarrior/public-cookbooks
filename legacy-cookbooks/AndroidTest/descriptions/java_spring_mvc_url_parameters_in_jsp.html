Abstract

Secure coding practices prescribe to not display or evaluate URL parameters.

Description

Displaying unescaped URL parameters leads to XSS vulnerabilities. Evaluating URL parameters allows for SpEL injection vulnerabilities. Escaping URL parameters does not solve this problem as there are valid expressions in the Spring Expression Languages that only contain legal XML or HTML characters. Instead it is recommended to never evaluate URL parameters.
