Abstract

Secure coding practices prescribe to configure channels to be secure.

Description

Spring-Security provides its own mechanisms to enforce secure channel usage. It is recommended to configure channels to make sure these mechanisms are used.

Correct code example:

public class WebSecConf extends WebSecurityConfigurerAdapter {
   ...
   @Override
   protected void configure(HttpSecurity http) throws Exception {
      http.requiresChannel().anyRequest().requiresSecure();
      ...
      }
   ...
}
