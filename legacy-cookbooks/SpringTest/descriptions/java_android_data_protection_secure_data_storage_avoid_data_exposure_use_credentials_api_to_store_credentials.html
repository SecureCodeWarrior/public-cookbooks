Abstract

Secure coding practices prescribe to use the Credentials API to store user credentials. This allows to automate future authentication in the application.

Description

It is recommended to store the user's credentials using the smart lock for passwords on Android. This can be done using the Credentials API.
Configure an android project by installing the Google Play services SDK in Android Studio by selecting Tools > Android > SDK Manager and selecting Google Play Services. Then adding the dependency: 
compile 'com.google.android.gms:play-services-auth:11.8.0'
to the gradle file.

Class information:

auth.api.credentials.CredentialsClient
    save(Credential)
    delete(Credential)

auth.api.credentials.Credential.Builder
    setName(String)
    setPassword(String)
    build()

Correct code example:

Credential credential = new Credential.Builder(email)
        .setPassword(password)  
        .build();
mCredentialsClient.save(credential).addOnCompleteListener(mListener);
If the call to CredentialsClient.save() is not immediately successful, the credentials might be new, in which case the user must confirm the save request. Resolve the ResolvableApiException with startResolutionForResult() to prompt the user for confirmation.
For more in depth explanation go to developers.google.com.
