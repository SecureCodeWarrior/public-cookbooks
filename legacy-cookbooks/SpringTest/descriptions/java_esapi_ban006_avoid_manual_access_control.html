Abstract

Out of Java best practices and coding guidelines, recommendations where abstracted that recommend the usage of OWASP Enterprise Security API (ESAPI). Only authorized routines from that library can be used for access control. As such, developers can not use standard java implementations to handle access control to URLs, functions, data, files, ... .

Description

It is not recommended to use the standard java implementation for access control as there is an OWASP ESAPI alternative. More specific, one should use an implementation of the AccessController interface, it provides access control for URLs, business functions, data, services, and files. Impementing classes of this interface are the DefaultAccessController and the ExperimentalAccessController.

Correct code example:

import org.owasp.esapi.reference.DefaultAccessController;

...
AccessController ac = DefaultAccessController.getInstance();
boolean allowed = ac.isAuthorizedForData(string, object);
