Abstract

Secure coding practices prescribe to use the cryptographic functions provided by the Spring Framework when possible.

Description

It is recommended to use the cryptographic functions provided by the Spring framework over the default Java implementations. This means that Java classes like Cipher, MessageDigest are not recommended since they have Java Spring alternatives.
Use the framework support for authorisation and authentication over manual asymmetric encryption, and use the password encoders over manual hashing and symmetric encryption.
