Abstract

Out of Java best practices and coding guidelines, recommendations where abstracted that recommend the usage of OWASP Enterprise Security API (ESAPI). Only authorized routines from that library should be used for saving Properties.

Description

It is not recommended to use the standard java implementation for saving properties as there is an OWASP ESAPI alternative. Concrete this means that the Properties class is banned. More specific, one should use an implementation of the EncryptedProperties interface, it provides an alternative with the same methods as the original Class but guarantees cryptographic security. Implementing classes of this interface are the DefaultEncryptedProperties and the ReferenceEncryptedProperties.

Correct code example:

import org.owasp.esapi.reference.crypto.DefaultEncryptedProperties;

...
EncryptedProperties p = new DefaultEncryptedProperties();
p.getProperty("prop1");
p.setProperty("prop2", "5");
p.load(in);
