Abstract

Secure coding practices prescribe to use EC to generate key pairs.

Description

Asymmetric encryption requires a pair of keys, a private and a public key. This enables authenticity on top of confidentiality in the encryption algorithm. Authenticity is a requirement for some secure communication channels. That means that asymmetric cryptography is typically used in a scheme for signatures or a key agreement and not for bulk encryption of data.
It is recommended to use EC over RSA for key pair generation. The ECGenParameterSpec class should be used when configuring the KeyPairGenerator class.

Correct code example:

SecureRandom secureRandom = new SecureRandom();
KeyPairGenerator keyPairGenerator;
keyPairGenerator = KeyPairGenerator.getInstance("EC");
keyPairGenerator.initialize(new ECGenParameterSpec("brainpoolP224t1"), secureRandom);
KeyPair keyPair = keyPairGenerator.generateKeyPair();
