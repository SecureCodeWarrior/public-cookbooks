Abstract

Secure coding practices prescribe to always use the latest version of dependencies. For sensei-secure-library this is version 2.6.

Description

When using older version of dependencies, it is likely these contain known vulnerabilities that have been resolved in newer versions. It is always recommended to use the latest version in your dependencies. In maven this can be done by specifying the version in de pom.xml file. Either explicitly define the version number you desire or use the latest keyword.

Correct code example:

<dependencies>
    <dependency>
        <groupId>com.securecodewarrior</groupId>
        <artifactId>sensei-secure-library</code>
        <version>latest</version>
    </dependency>
    ...
</dependencies>
