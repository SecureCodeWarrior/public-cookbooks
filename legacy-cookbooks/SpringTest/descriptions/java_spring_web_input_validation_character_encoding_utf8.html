Abstract

Secure coding practices prescribe to set the character encoding explicitely.

Description

Secure coding practices prescribed to specify proper character sets, such as UTF-8, as part of the input validation routines. This is because special (dangerous) characters might be encoded in different ways. For example the characters %c0%ae%c0%ae/, which means ../ (dot-dot-slash) can still be used for directory traversal if the input validation checks for dots and slashes but not these special characters.

Correct code example:

<filter>
   <filter-name>encodingFilter</filter-name>
   <filter-class>org.springframework.web.filter.CharacterEncodingFilter</filter-class>
   <init-param>
      <param-name>encoding</param-name>
      <param-value>UTF-8</param-value>
   </init-param>
   <init-param>
      <param-name>forceEncoding</param-name>
      <param-value>true</param-value>
   </init-param>
</filter>
<filter-mapping>
   <filter-name>encodingFilter</filter-name><
   <url-pattern>/*</url-pattern>
</filter-mapping>
