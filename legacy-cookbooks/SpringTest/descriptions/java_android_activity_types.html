Activity Types

Based on tags and attributes used in activities, four types can be specified: private, public, partner and in-house.
  1. Private activity: Only accepts intents from the same app.
    <activity android:exported="false" ... />
    The android:launchMode and android:taskAffinity should not be set for the private Activity. (Read why: Page 53)
  2. Public activity: accepts implicit and explicit intents from other apps and the OS.
    Security implications for public activities: This means that malware will also be able to send intents. It is recommended to use partner or in-house activities over public activities. However for the OS to launch the app at least the main activity needs to be a public activity (usually called MainActivity). 
    <activity android:exported="true" ... />
...
<intent-filter>
   <action android:name="com.examples.ACTION"/>
   <category android:name="android.intent.category.XXX"/>
</intent-filter>
  3. Partner activity: only accepts explicit intents from a defined app white-list. 
    <activity android:exported="true" ... />
    The white-list with approved apps needs to be defined in the Activity itself (Sensei help is available there)
  4. In-house activity: Only accepts explicit intents from apps with the right permission. 
    <activity android:exported="true"
   android:permission="com.senseissecurity.activity.PERMISSION" .../>
