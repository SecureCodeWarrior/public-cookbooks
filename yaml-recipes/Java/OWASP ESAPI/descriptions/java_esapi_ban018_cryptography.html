Abstract

Out of Java best practices and coding guidelines, recommendations where abstracted that recommend the usage of OWASP Enterprise Security API (ESAPI). Only authorized routines from that library should be used for for cryptography.

Description

It is recommended not to use the standard java implementation for encoding text as there is an OWASP ESAPI alternative.
Concretely, the following classes are banned:
  • Cipher
  • KeyGenerator
  • KeyPairGenerator
  • KeyAgreement
  • KeyFactory
  • SecretKeyFactory
  • MessageDigest
  • Signature
  • Timestamp
More specific, one should use an implementation of the Encryptor interface, it provides an alternative with the same methods as the original Class but guarantees cryptographic security. An implementing class of this interface is the JavaEncryptor.

Correct code example:

import org.owasp.esapi.reference.JavaEncryptor;
...
PlainText pt = JavaEncryptor.getInstance().decrypt(ciphertext);;
