Logging any issues is considered a best practice. Making a logger available in every class reduces the threshold to use logging.


Logging errors, warnings and other debugging information ensures that the information does not get lost. Centralised logging with multiple error levels enables you to have a clear overview of any issues in the application.


Before 
public class Logging {
    ...
}
}

After 
import java.util.logging.Logger;

public class Logging {

    Logger logger = Logger.getLogger(Logging.class.getName());

    ...
}
}
