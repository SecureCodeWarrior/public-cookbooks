Bitwise operators are not used often in conditionals. Perhaps the intended operator here is the logical operator.


Using the incorrect operator can lead to a NullPointerException, unintended behaviour in for example short circuit evaluation, and even bypassing checking or branching logic.


Before 
if (object != null & object.myBoolMethod()) {
  // if object is null, object.myBoolMethod() is still evaluated, causing a NullPointerException
}

After 
if (object != null && object.myBoolMethod()) {
  // if object is null, object.myBoolMethod() is not evaluated due to short circuit evaluation
}
