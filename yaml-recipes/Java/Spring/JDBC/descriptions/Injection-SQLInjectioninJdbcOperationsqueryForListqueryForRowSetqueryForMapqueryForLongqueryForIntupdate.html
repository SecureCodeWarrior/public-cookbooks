Dynamically creating queries by means of concatenating (untrusted) user input puts the application at risk of SQL injection. An attacker could insert malicious input to obtain or modify data. Use parameterized queries to prevent SQL injection.
Before:jdbcOperations.update("delete from users where username = '" + username + "'");
After:jdbcOperations.update("delete from users where username = ?", username);
