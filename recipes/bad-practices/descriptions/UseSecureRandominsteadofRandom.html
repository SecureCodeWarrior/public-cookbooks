Abstract

Secure coding practices prescribe that the use of java.util.Random should be avoided. java.security.SecureRandom should be used instead.

Description

Using Random can lead to predictable randomness. Examples:

  • generated secrets
  • CSRF Tokens
  • Login and password reset tokens
Before: 
Random rnd = new Random();
After: 
SecureRandom rnd = new SecureRandom();
