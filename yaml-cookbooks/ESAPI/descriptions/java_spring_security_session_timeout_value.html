Abstract

Secure coding practices prescribe to set session timeout to the minimal value possible.

Description

Session timeout define action window time for a user thus this window represents, in the same time, the delay in which an attacker can try to steal and use a existing user session. For this, it is best practice to :
  1. Set session timeout to the minimal value possible depending on the context of the application.
  2. Avoid "infinite" (value 0) session timeout .
A recommended value is around 10 minutes.

Correct code example:

<session-config>
   <session-timeout>10</session-timeout>
</session-config>

Violating this guideline can cause

Web vulnerabilities

  • Session Handling - Insufficient Session Expiration. Learn more
