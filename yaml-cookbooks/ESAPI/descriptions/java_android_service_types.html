Service Types

Based on tags and attributes used in services, four types can be specified: private, public, partner and in-house.
  1. Private Service: Can only receive intents from the App itself. 
    <service android:exported="false" ... />
  2. Partner Service: Can only receive explicit intents from a white-list of partner apps.
    Does not define an intent filter and does not use the attribute android:permission. 
    <service android:exported="true" ... />
  3. In-House Service: Can only receive explicit intents from apps that have the right permission.
    Does not define an intent filter. 
    <service android:exported="true"
   android:permission="com.senseisecurity.service.inhouseservice.SERVICE_PERMISSION" />
  4. Public Service.
    Security implications for public services: Public services as may give unauthorized applications access to private information. Public services have the android:exported attribute set to true and define an intent filter, this means that they can be used by any applications, so they can also be used by malware. Instead developers should use In-house services, so that the apps require permission and the user has control over what app can potentially access his data. Or developers can use partner services so that they can whitelist apps to use their service.
