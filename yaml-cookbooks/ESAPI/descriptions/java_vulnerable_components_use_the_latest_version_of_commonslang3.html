Abstract

Secure coding practices prescribe to always use the latest version of dependencies. In particular, make sure the commons-lang3 library often used for output escaping is up to date.

Description

When using older version of dependencies, it is likely these contain known vulnerabilities that have been resolved in newer versions. It is always recommended to use the latest version in your dependencies. In maven this can be done by specifying the version in de pom.xml file. It is recommended to use the latest keyword rather than specifying the version manually.

Correct code example:

<dependencies>
    <dependency>
        <groupId>org.apache.commons</groupId>
        <artifactId>commons-lang3</code>
        <version>latest</version>
    </dependency>
    ...
</dependencies>
