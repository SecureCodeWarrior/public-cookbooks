Abstract

Secure coding practices prescribe that certain libraries cannot be used because they are unsafe from a security perspective.

Description

It is recommended to avoid using any method that are associated with executing dynamically loaded code at runtime. This often allows for injection vulnerabilities.
The following classes and methods are recommended to avoid:
package javax.script.Compilable;
   CompiledScript compile(Reader script);
   CompiledScript compile(String script);

package java.lang.Runtime;
   void load(String filename);
   void loadLibrary(String libname);
   Process exec(...); //any overloaded

package java.lang.System;
   static void loadLibrary(String libname);

package java.beans.XMLDecoder;

package java.net.URLClassLoader;

package javax.script.ScriptEngineManager;
