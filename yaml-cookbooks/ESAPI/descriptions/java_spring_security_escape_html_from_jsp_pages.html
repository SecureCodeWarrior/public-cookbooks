Abstract

Secure coding practices prescribe to set the context parameter defaultHtmlEscape to true in the web.xml file.

Description

In Java Spring Security it is possible to escape the html from JSP pages generated by <form> tags. This closes off a lot avenues for XSS attacks, and can be done automatically in several ways.
The recommended way is to escape automatically for the entire application by setting the context parameter defaultHtmlEscape in the web.xml file to true:

Correct code example:

<context-param>
   <param-name>defaultHtmlEscape</param-name>
   <param-value>true</param-value>
</context-param>

Violating this guideline can cause

Web vulnerabilities

  • Cross-Site Scripting (XSS) - Stored Cross-Site Scripting. Learn more
  • Cross-Site Scripting (XSS) - Reflected Cross-Site Scripting. Learn more
  • Cross-Site Scripting (XSS) - Mixed server and client side rendering. Learn more
