Abstract

Secure coding practices prescribe to use strong password policies.

Description

Many systems use passwords to authenticate the user. It is recommended to use strong password policies to prevent password guessing, brute forcing and other attacks. A good policy includes:
  • A good minimum and maximum password length. Short passwords have insufficient entropy and are more easily guessed. Long password lengths can lead to DOS. A reasonable password length is between 10 and 160 characters.
  • Do not limit the character set. This is often done to prevent SQL injection or other input validation attacks. Protect against these attacks with proper input escaping instead of limiting the character set. A limited character set facilitates password brute forcing and guessing.
  • Passwords may not contain personal information. This makes password guessing easier.
  • Passwords may not be reused. If passwords, protected or unprotected, leak from a third party, reusing of passwords will make it possible for an attacker to successfully authenticate with the leaked credentials.
  • Expiration of passwords is recommended in sensitive context.
  • Display a generic message when any of the credentials are incorrect. Do not specify whether the username or the password is incorrect to prevent username enumeration.
  • It is recommended to force the user of your application to create a strong password. It is recommended to force a user to use at least:
    • One upper case character
    • One lower case character
    • One numeric character
    A common practice is to indicate password strength to the user through a combination of measures such as length and sufficient entropy in the used characters.
