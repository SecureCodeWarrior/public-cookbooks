Abstract

Secure coding practices prescribe for logging and error handling.

Description

It is recommended not to print stack traces or log events to stdout/stderr as this will leak information about the code to malicious users. Instead events can be logged with a logger that references the relevant class.
Avoid string concatenation, but use placeholders in your log Strings instead.

Correct code example

class ValueSetter{
    private Logger LOGGER = LoggerFactory.getLogger(ValueSetter.class);
    public void setValue(int newValue){
        try {
           ...
        } catch (Exception e){
            LOGGER.error("Failed to set value to {}.",  newValue);
        }
        LOGGER.info("Set value to {}.",  newValue);
    }
}
